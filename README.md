# SIPAVideo_Analysis
The python code "track_thrombotic_islands.py" is to track the SIPA islands from their creation, to expansion, over time by tracking their border, distinguished by the brightness change, over time. Python code "SIPA_Island_analysis.py" analyse and plot graphs based on the data generated by the "track_thrombotic_islands.py". Python code "SIPA_totalBrightnessChange.py" only records the total frame brightness change over time (it is not very sensitive and does not seem to be able to capture the SIPA island creation and progression over time).   

Our original videos ".avi" seem to have inconsistent framerate throughout the video, which causes problems when processing them. For example, when the process gets to where the framerate changes, cv2 cannot handle it and goes back to the first frame, which causes repeatition in the final results. Therefore, it is needed to change the encoding of the video using:
<pre>
  <code>
    ffmpeg -i your_video.avi -c:v libx264 -preset slow -crf 18 -pix_fmt yuv420p -vsync cfr -r 30 output_fixed.mp4
    </code>
</pre>
